# Goodminton 学员反馈模板

这个模板服务于学生页，不是单纯的档案表。每个字段都要能回到反馈系统：课堂观察、学生感受、家庭作业、比赛复盘、能力矩阵、技能树、成就。

## 建档方式

1. 复制 `data/student-template.json`。
2. 改名为 `data/students/{studentId}.json`。
3. `studentId` 使用拼音，例如 `xue-meijiao`。
4. `accessCode` 单独设置，不和学生姓名一起公开。
5. 真实学员 JSON 不提交到 GitHub，更新后同步到 Vercel 环境变量。

## 页面字段

`level`

内部等级，可继续写 `C1`、`C2`、`B1`、`A1`。学生页会换算成 `白金 / 钻石 / 大师 / 王者`。

`progress`

总进度，0-100。它会影响顶部进度显示，也作为等级换算的辅助依据。

`abilityMatrix`

固定六项：

- 身体
- 心理
- 比赛
- 技术
- 战术
- 步法

每项 0-100。这个是教练主观评估加课堂证据，不追求一次精确，后面每几节课校准。

`skillTree`

固定大类：

- 发接发
- 前场
- 中场
- 正手后场
- 头顶后场
- 反手后场
- 步法
- 心理
- 身体
- 战术

每个节点只用三种状态：

- `done`：已经稳定掌握
- `active`：当前训练中
- `locked`：还没开始或不稳定

柱形图高度由 `done + active` 的数量决定。比如“前场第 5 档”，就是前 4 个节点 `done`，第 5 个节点 `active`，后面 `locked`。

`lessonSummary`

当前课后反馈区。学生自己填写 `studentReflection`，教练填写 `coachNote` 和 3 条家庭作业。家庭作业只保留这一处，避免页面重复。

`lessonHistory`

上课日志，按日志方式滚动显示。每节课至少要有：

- date
- title
- mainContent
- coachNote
- studentNote
- homeworkDone
- homeworkTotal

`matchReview`

比赛复盘由学生自己填。默认可以留空，页面提供输入框：

- 比赛 / 对手
- 比分
- 有效的地方
- 待改进的点
- 积累的经验

`achievements`

成就和勋章要来自真实行为，不做装饰。默认五类：

- 首条复盘：完成第一次课后总结
- 作业连续完成：连续完成家庭作业
- 技术采集者：技能树产生稳定掌握点
- 步法底盘：启动、移动、回动、调整形成连接
- 平分冷静：比赛压力下能做清晰选择

## 调整数值的最短格式

可以直接这样给：

```text
美姣：技术 65，步法 68，身体 75，心理 85，比赛 72，战术 50；
发接发第3档，前场第5档，正手后场第4档。
```

然后只改对应 JSON，并同步线上环境变量。
